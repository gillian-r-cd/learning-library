# 学员端 UI/UX Impeccable 审计与改造方案

## 0. 结论摘要

当前学员端的底层机制已经很有想象力：英雄之旅、Narrator、Companion、道具、招式集、能力地图、Manifesto、点数衰减与恢复，这些都足够支撑一个“学习者在情境里变强”的产品。

但现有 UI 的表达还偏 **AI 工具 / 调试面板 / 数据看板**。它把很多系统真实能力暴露出来了，却没有把这些能力翻译成学员能感到“我进入了一个世界”“我刚刚真的变强了”“我想再来一轮”的体验。

如果用一句话定义改造目标：

> 把学员端从“AI 评估控制台”改成“一个会记住你、奖励你、把你的成长变成故事的学习剧场”。

优先级最高的不是换颜色，而是重排体验节奏：

1. 开场要像进入一幕剧，而不是看到一段普通聊天气泡。
2. 回答后要有即时反馈，不只是 Narrator 文本和分数数字变化。
3. 加分、解锁、获得招式、掉落道具、完成挑战，都要有 “moment”。
4. 右侧看板要从调试信息转成成长仪表。
5. 开发者概念和 AI 内部名词要从学员主路径移走。

## 1. Impeccable 评估框架

这里的 “impeccable” 不是单纯“好看”，而是五个维度同时成立：

| 维度 | 问题 | 对本项目的含义 |
| --- | --- | --- |
| Immersion 沉浸 | 学员是否觉得自己进入了一个具体场景？ | 开场、道具、人物、旁白要像故事，不像日志。 |
| Motivation 动机 | 学员是否知道为什么继续下一轮？ | 进度、下一位伴学、下一招、下一格能力地图要清楚召唤。 |
| Progress 获得感 | 学员是否感到自己刚刚变强？ | 加分、招式、能力格、Manifesto 要在主路径里被看见。 |
| Clarity 清晰 | 学员是否知道现在该做什么？ | 输入框架、当前挑战目标、反馈重点要明确。 |
| Craft 精致 | 是否减少 AI 味、工程味和占位感？ | 去掉 `Judge`、`turn #`、`free_text` 等内部语言，换成产品语言。 |

## 2. 当前 UI 的主要问题

### 2.1 主界面像调试台，不像学习剧场

`LearnerSession` 顶栏现在塞了大量入口：

- 道具箱
- 招式集
- 伴学库
- 能力地图
- 点数
- turn
- 消息数

这些功能本身都重要，但被并排放在顶栏后，学员第一眼看到的是“系统控件”，不是“我在第几幕、面临什么任务、刚刚获得了什么”。

建议：

- 顶栏只保留当前挑战标题、章/阶段、核心进度和一个“我的成长”入口。
- 道具、招式、伴学、能力地图合并进右侧或底部 “成长背包”。
- `turn #` 和 `msgs` 只保留给开发模式，默认不展示。

### 2.2 AI 味最重的位置

| 位置 | 当前问题 | 建议方向 |
| --- | --- | --- |
| `Judge 最近一次输出` | 直接展示 JSON，是典型 Agent 调试 UI。 | 移到开发者抽屉；学员只看“这次系统看见了什么”。 |
| `Evidence 最近 10 条` | `action_id`、`d1:good`、raw evidence 像评分后台。 | 改成“本轮被看见的能力”或“成长记录”。 |
| `ResponseFrameRenderer` | 显示 `free_text` / `form` 等 schema kind。 | 换成“自由作答”“诊断表”“选择一步行动”。 |
| 顶栏 `turn #` | 是运行时概念。 | 改为“第 3 次尝试”或完全隐藏。 |
| 中英并列标签 | 很多组件像内部 demo。 | 学员端默认中文叙事，英文只留后台或开发模式。 |
| `complexity: low/medium/high` | 工程标签。 | 改成“入门 / 进阶 / 高压”。 |

### 2.3 获得感节点存在，但没有被放大

已有机制：

- 加分：`pointsEarned`、`newTotal`、`effectiveTotal`
- 解锁伴学：`newUnlocks`
- 获得招式：`AWARD_SIGNATURE_MOVE`
- 道具掉落：`DROP_ARTIFACT`
- 完成挑战：`challenge_completed`
- 章节宣言：`Manifesto`
- 能力地图：`MasteryHeatmap`

但 UI 大多把它们作为普通系统消息、右栏数字或抽屉列表呈现。缺少一套“事件分级与过场表达”。

建议新增统一的 **Learning Moment System**：

| 事件等级 | 例子 | 表达方式 |
| --- | --- | --- |
| Micro 微反馈 | +3 分、某维度 good | 输入后 500ms 内轻动效、分数飞入、能力小徽章闪一下。 |
| Meso 中反馈 | 获得招式、道具掉落、伴学发言 | 主对话流插入高光卡片，有短音效/动效空间。 |
| Macro 大反馈 | 完成挑战、章节完成、生成 Manifesto | 全宽过场卡或轻量 modal，形成“阶段结算”。 |

## 3. 新体验愿景

学员端应从“三栏工具页面”变成“三层学习剧场”：

### 3.1 第一层：舞台

左侧主区域是舞台，承载：

- 开场背景
- Narrator 旁白
- 学员行动
- 道具出现
- 伴学插话
- 高光奖励

它应该像一条叙事时间线，而不是普通 chat log。

### 3.2 第二层：行动台

底部输入区是行动台，承载：

- 自由作答
- 诊断表
- 选择题
- 多选策略
- 结构化任务

它应该让学员感到“我知道下一步该做什么”，而不是“我在给 AI 发 prompt”。

### 3.3 第三层：成长仪表

右侧是成长仪表，承载：

- 当前阶段问题
- 下一格能力地图
- 下一招可能获得什么
- 下一位伴学还差多少
- 本章 Manifesto 素材进度

它不应该默认展示 Judge JSON 和 evidence log。

## 4. 全局视觉语言调整

### 4.1 从“深色控制台”到“夜间学习剧场”

当前深色底色可以保留，但要减少纯面板堆叠感。

建议视觉关键词：

- 背景：深蓝黑 + 微弱径向光，像夜间舞台。
- 主舞台：左侧对话区有更明显纵向时间线或“场景层”。
- 右侧：从卡片堆叠改成“成长卷轴 / 仪表盘”。
- 高光色：保留 accent 蓝，但奖励类使用金色、绿色、紫色分层。

建议 token：

```txt
stage-bg: radial-gradient(circle at 30% 0%, rgba(106,166,255,.14), transparent 42%), #0b0f19
moment-gold: #ffd166
moment-purple: #b892ff
moment-green: #7bd88f
ink-muted: #9aa7bd
panel-glass: rgba(19,25,41,.78)
```

### 4.2 建立“moment card”体系

新增统一视觉层级：

| 类型 | 用途 | 视觉 |
| --- | --- | --- |
| Scene Card | 开场、换挑战 | 大卡、渐变边框、章节/阶段 badge、淡入。 |
| Reward Card | 加分、招式、伴学解锁 | 金色/绿色高光、数字跳动、短文案。 |
| Artifact Card | 道具掉落 | 类文件卡、纸张质感、打开按钮。 |
| Reflection Card | Manifesto / 金句 | 引号、署名、柔和光晕。 |
| Scaffold Card | 支架帮助 | 不叫“scaffold”，改叫“给你一个抓手”。 |

## 5. 分组件改造方案

### 5.1 `LearnerSession` 顶栏

当前问题：

- 信息太多。
- 功能入口像后台工具。
- `msgs` 和 `turn #` 是内部状态。

建议改成：

```txt
[第 2 章 · 试炼] 准备度诊断：第一次对话        [旅程 35%] [成长背包]
```

具体调整：

- `complexity` 映射为中文难度：`low=入门`、`medium=进阶`、`high=高压`。
- `turn #` 显示为“第 N 次尝试”，或只在开发模式显示。
- 删除 `msgs.length` 展示。
- 将道具箱、招式集、伴学库、能力地图、点数合并成一个 `成长背包` 按钮，打开后有 tabs。

如果短期不重构抽屉，也可以先视觉分组：

- 左侧：当前挑战信息。
- 右侧：一个主按钮“成长背包” + 一个点数胶囊。
- 其他入口移到右栏卡片。

### 5.2 开场背景 `challenge_opening`

当前问题：

- 只是带 accent 左边框的普通 Narrator 气泡。
- 没有明显“开幕”体验。

建议：

- 单独做 `SceneOpeningCard`。
- 全宽或接近全宽。
- 顶部显示：章节、弧阶段、阶段问题。
- 正文使用更大的行距和更强层级。
- 首次进入时淡入 + 背景微光扫过。

示例文案结构：

```txt
第二章 · 试炼
你正在面对的问题：当默认做法失效时，你还能读懂什么？

【场景开幕】
你推开小会议室的门……

本幕目标：先观察，再判断，不急着给方案。
```

动画建议：

- `fade-up 360ms`
- 背景 `soft-shine 900ms`
- 弧阶段 badge 延迟 120ms 出现

注意：动画要服务沉浸，不要像营销弹窗。

### 5.3 Narrator 气泡

当前问题：

- `Narrator` 这个词 AI 味强。
- 普通文本气泡太像聊天机器人回复。

建议：

- 学员端显示为“旁白”或“场景”。
- Narrator 的普通反馈分三种视觉：
  - 普通推进：轻卡片。
  - 支架帮助：抓手卡。
  - 收束挑战：结算前置卡。
- 去掉英文 label。

建议名称映射：

```txt
Narrator -> 旁白
Companion -> 伙伴 / 伴学姓名
System milestone -> 挑战完成
System unlock -> 新能力解锁
```

### 5.4 学员气泡

当前问题：

- 右对齐普通聊天气泡可以接受，但结构化表单提交只显示“已提交：xxx”会弱化学员行动。

建议：

- 自由文本：保持对话气泡。
- 表单提交：显示为“我的行动卡”，列出关键字段摘要。

示例：

```txt
我的诊断
对象：陈悦
能力：中
意愿：低
证据：能复述流程，但回避承诺排期
```

这样学员能回看自己的思路，而不是看到“已提交表单”。

### 5.5 `ResponseFrameRenderer`

当前问题：

- 显示 `frame.kind`，工具味强。
- 表单样式仍像后台表单。
- busy 状态是 `...`。

建议：

- 隐藏 schema kind，改用人话标签。
- 根据 kind 显示行动提示：
  - `free_text`：写下你的判断。
  - `single_choice`：先做一个选择。
  - `multi_choice`：选择你看到的信号。
  - `form`：把判断拆开。
- 提交按钮文案更贴近动作：`提交判断`、`进入下一步`、`交给旁白评估`。
- busy 状态改为：`旁白正在观察你的判断…`
- 对表单增加“完成度提示”：`3/4 已填写`。

更 Aha 的方向：

- 当 Judge 切换到结构化框架时，不要突然换表单，而是显示一句：

```txt
你似乎卡在“能力 vs 意愿”的拆分上。
这次我们换成诊断表，只看四件事。
```

这会让结构化输入像认知支架，而不是系统表单。

### 5.6 加分体验

当前问题：

- 分数在顶栏数字变化，点数明细在 modal。
- 学员未必感到“刚刚这轮哪里做对了”。

建议新增 `PointsMoment`：

触发条件：

- 每轮 `pointsEarned > 0`

表现：

- 对话流中 Narrator 后插入小型奖励条。
- 点数从 `+0` 跳到 `+N`。
- 展示一个主因，而不是完整公式。

示例：

```txt
+6 分
因为你把“能力线索”和“意愿线索”分开了。
```

同时右上点数胶囊做 600ms 高光：

```css
score-pulse: scale(1) -> scale(1.08) -> scale(1)
```

### 5.7 挑战完成过场

当前问题：

- `milestone` 是普通系统消息。
- 完成挑战没有“关卡结算”体验。

建议新增 `ChallengeCompleteCard`：

内容：

- 挑战名
- 本轮学到的核心动作
- 获得分数
- 点亮的能力格
- 下一幕标题

示例：

```txt
挑战完成
你完成了「第一次对话」

你刚刚建立起：信号 → 判断 → 风格选择
+8 分 · 能力地图点亮 1 格

下一幕：小陈的周报
```

动画：

- 中央卡片从对话流中展开，不要强制全屏太久。
- 1.2 秒内完成，允许跳过。

### 5.8 章节完成与 Manifesto

当前问题：

- Manifesto 是右栏面板。
- 生成时没有强仪式感。

建议：

- 章节完成时出现 `ManifestoRevealCard`。
- 用学员原话做引用动效。
- 标题从“我的宣言 / Manifesto”改成“这一章，你留下的话”。

示例：

```txt
这一章，你留下了一句话

“我不能再只看一个人安不安静，而要看他面对具体任务时露出的能力和意愿。”

系统把你这一章的判断方式编成了一段宣言。
[查看我的宣言]
```

空状态改造：

当前：

```txt
完成一整章后，这里会用**你自己的原话**合成...
```

问题：`**` 在普通 div 中不会渲染 Markdown。

建议：

```txt
完成一整章后，这里会收集你说过的关键判断，
编成一段由你署名的章节宣言。
```

### 5.9 招式集

当前问题：

- 侧栏列表已经清楚，但获得瞬间不够强。
- “招式”本身很有产品想象力，应作为核心爽点。

建议：

获得招式时出现 `SignatureMoveMoment`：

```txt
你打出了一招

分任务诊断
不给人贴整体标签，对同一人在不同任务上分别落档。

首次触发来自你的这句话：
“……”
```

视觉：

- 武侠/工具隐喻都可以，但不要游戏廉价化。
- 用金色细线、印章感、招式名大字。

侧栏调整：

- 已获得招式按 tier 分组：初识、娴熟、立派。
- 未获得招式显示“下一次怎样打出来”，但不要透露太像攻略。
- 每张卡显示最近一次触发，而不是只显示首次。

### 5.10 伴学库

当前问题：

- 有丰富 persona，但列表像系统资料表。
- 伴学在主体验里不够像“关系”。

建议：

解锁伴学时出现 `CompanionUnlockMoment`：

```txt
有人加入了你的旅程

Elena
她会在你卡住时，把抽象判断翻译成可观察的线索。

关系：Lv.1 初识
[听她说一句]
```

伴学气泡建议：

- 不同 companion type 有不同气泡风格：
  - guide：蓝绿色，稳定。
  - competitor：红/橙，挑战。
  - traveler：暖黄，同行。
  - adversary：暗红，质疑。
  - case_pack：纸张/资料卡。
- 气泡 label 只显示姓名，不显示 `companion_type`。

侧栏调整：

- “未解锁 · 差 X 分”改为更有召唤感：

```txt
再完成 2 次高质量判断，她就会出现。
```

### 5.11 道具箱与道具掉落

当前问题：

- 道具箱像资源库。
- 道具掉落没有“发现线索”的感觉。

建议：

道具掉落时出现 `ArtifactDropMoment`：

```txt
你发现了一份新材料

王磊 · 员工档案卡
里面可能藏着判断“能力/意愿”的线索。
[打开查看]
```

道具箱调整：

- 按“本章材料 / 历史材料”分组。
- 道具卡显示一句“为什么重要”，可从 `narrator_intro` 或 artifact summary 派生。
- `fields` 类型做成档案卡质感；`table` 做成报表质感；`narrative` 做成信件/邮件质感。

### 5.12 能力地图

当前问题：

- heatmap 功能清楚，但像数据表。
- “我变强了”的感受不够。

建议：

- 主入口显示一句总结：

```txt
你已经点亮 5/18 格，最稳的是「读懂准备度」。
```

- modal 内表格可保留，但视觉从 table 改为地图：
  - 每个 core action 是一条路径。
  - low/medium/high 是三个节点。
  - good/medium/poor 用节点状态表现。

短期折中：

- 保留表格结构，但加顶部 “下一格”：

```txt
下一格建议：在高压情境下练「判断准确性」
```

### 5.13 点数明细

当前问题：

- 公式完整，但像审计后台。
- `Raw / Effective / with decay` 对学员不够自然。

建议：

分两层：

1. 学员层：本轮为什么得分、累计成长。
2. 详情层：公式、衰减、每维评分。

主文案替换：

```txt
当前战力：128
近期仍然稳固：114
```

不要默认说 “with decay”，改为：

```txt
有效点数会随着长期不练而变淡，重新练习可以恢复。
```

### 5.14 ProgressPanel

当前问题：

- 是清单，不是旅程地图。

建议：

- 当前章展开，其它章折叠。
- 当前挑战用“所在节点”强标记。
- 完成章不要 opacity 变灰，而是像已盖章。
- 里程碑 summary 做成章卡片底部一句话。

更 Aha 的方向：

```txt
你正在第 3 个节点：
从“观察信号”走向“做出风格选择”
```

### 5.15 右侧 Judge / Evidence 区

当前问题：

- 这是最重的 AI 味来源。

建议默认替换为 `GrowthFeedbackPanel`：

```txt
这轮系统看见了：
✓ 你引用了具体行为线索
△ 判断依据还可以再补一层

下一轮试试：
把“能力”和“意愿”分开写。
```

保留一个小链接：

```txt
查看评估详情
```

点开后再出现原 Evidence / Judge JSON。对于普通学员，不默认出现。

## 6. 信息架构重排

建议学员端最终结构：

```txt
┌────────────────────────────────────────────────────────────┐
│ 第2章 · 试炼     第一次对话                 旅程 35%  成长背包 │
├──────────────────────────────────────┬─────────────────────┤
│                                      │ 当前阶段             │
│  Stage / Conversation Timeline       │ 本阶段问题           │
│  - SceneOpeningCard                  │                     │
│  - Narrator                          │ 下一步建议           │
│  - LearnerActionCard                 │                     │
│  - PointsMoment                      │ 成长摘要             │
│  - ArtifactMoment                    │ - 下一招             │
│  - Companion                         │ - 下一位伴学          │
│  - ChallengeCompleteCard             │ - 下一格能力地图       │
│                                      │                     │
├──────────────────────────────────────┴─────────────────────┤
│ ResponseFrameRenderer：行动台                               │
└────────────────────────────────────────────────────────────┘
```

## 7. 动效原则

动效要克制，但关键 moment 要有明确反馈。

### 7.1 必须有动效的节点

| 节点 | 动效 |
| --- | --- |
| 开场 | fade-up + soft shine |
| 提交后等待 | 输入区轻微 breathing / “旁白正在观察” |
| 加分 | 数字 count-up + 点数胶囊 pulse |
| 获得招式 | 卡片 stamp-in |
| 道具掉落 | 文件卡 slide-in |
| 伴学解锁 | avatar reveal |
| 挑战完成 | completion card expand |
| Manifesto 生成 | quote reveal |

### 7.2 不建议做的动效

- 全屏烟花。
- 长时间不可跳过动画。
- 每轮都过度庆祝。
- 和学习语境无关的 emoji 爆炸。

## 9. 推荐落地顺序

### Phase 1：去 AI 味与信息层级

目标：不大改数据结构，先让学员端不像后台。

改动：

- 顶栏重排，隐藏 `msgs`、`turn #`、英文标签。
- `Judge JSON` 和 `Evidence` 默认收起到开发者详情。
- `ResponseFrameRenderer` 隐藏 schema kind，替换人话文案。
- `Narrator` 改为“旁白”，系统事件改为 moment 名称。

收益：最快降低 AI 味。

### Phase 2：主路径 moment 卡

目标：让加分、解锁、完成挑战有获得感。

改动：

- `PointsMoment`
- `ArtifactDropMoment`
- `SignatureMoveMoment`
- `CompanionUnlockMoment`
- `ChallengeCompleteCard`
- `ManifestoRevealCard`

收益：把已有系统事件变成体验高点。

### Phase 3：右侧成长仪表

目标：右栏从组件堆叠变成学习导航。

改动：

- `GrowthRail`
- 下一格能力地图
- 下一招提示
- 下一位伴学召唤
- 本章 Manifesto 素材进度

收益：学员知道为什么继续。

### Phase 4：视觉系统升级

目标：从深色控制台变成学习剧场。

改动：

- 新 token：stage、moment、glass、gold、purple。
- card 体系升级：scene / reward / artifact / reflection / help。
- 微动效。
- 道具内容做类型化质感。

收益：整体质感和想象力提升。

## 11. 验收标准

### 11.1 去 AI 味

- 学员默认界面不出现 `Judge`、`JSON`、`free_text`、`form`、`turn #`、`action_id`、`d1:good`。
- 系统内部信息仍可在开发者详情中查看。
- 学员端中英并列标签减少 80% 以上。

### 11.2 学习体验

- 每次提交后 1 秒内有明确反馈：正在评估 / 已获得 / 下一步。
- 加分有可见动画或高光。
- 挑战完成有过场卡。
- 章节完成有 Manifesto reveal。
- 获得招式和伴学解锁进入主对话流，而不只在侧栏更新。

### 11.3 想象力

- 开场明显像一幕场景，而不是普通聊天。
- 道具像线索，不像附件。
- 招式像能力收藏，不像标签。
- 伴学像关系，不像功能插件。
- 能力地图像旅程路径，不像二维表。

## 12. 风险与边界

1. 不要为了 Aha 牺牲清晰度。学员仍然要知道该做什么。
2. 不要让动画拖慢学习节奏。所有过场都应可跳过或短于 1.5 秒。
3. 不要把所有反馈都游戏化。不同事件要分级，避免廉价奖励疲劳。
4. 保留开发者诊断能力，但默认对学员隐藏。
5. 当前已有大量测试依赖 `data-test-id`，视觉改造应保留关键测试 id 或同步更新 E2E。

## 13. 推荐的第一版改造范围

第一版不建议一口气做完整视觉重构。建议先做：

1. 顶栏简化。
2. 开场 SceneOpeningCard。
3. ResponseFrameRenderer 去 schema 味。
4. PointsMoment。
5. ChallengeCompleteCard。
6. Judge/Evidence 移入开发者详情。

这 6 项能最快改变学员的第一感受，也最直接服务学习体验。
